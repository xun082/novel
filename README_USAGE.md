# 并行小说生成系统

基于 RWKV 模型的智能小说创作平台，支持多线程并行生成，自动扩写和优化内容。

## 功能特点

### 🚀 并行生成架构
- **多小说并行**: 同时生成多个不同的小说版本（可配置1-8个）
- **章节并行**: 每个小说的所有章节同时生成
- **智能调度**: 自动管理并发请求，提高生成效率

### 📝 四步生成流程

1. **生成大纲阶段**
   - 并行生成多个小说大纲
   - 包含完整的标题和章节概要
   - 每个大纲独立生成，互不干扰

2. **内容生成阶段**
   - 基于大纲并行生成每章正文
   - 目标字数：500-1000字/章
   - 实时显示生成状态

3. **智能扩写阶段**
   - 自动检测内容长度
   - 对少于300字的章节自动扩写
   - 确保每章内容完整丰富（800-1000字）

4. **过渡句生成**
   - 为相邻章节生成自然过渡
   - 提升阅读连贯性
   - 增强故事流畅度

## 使用方法

### 1. 配置参数

在界面上设置以下参数：

- **小说类型**: 例如"玄幻"、"仙侠"、"科幻"、"都市"等
- **章节数量**: 5-50章（建议15章）
- **并行数量**: 1-8个（建议3个）

### 2. 开始生成

点击"开始生成"按钮，系统将自动执行四步流程：

```
步骤1: 生成大纲（25%进度）
  ↓
步骤2: 生成章节内容（60%进度）
  ↓
步骤3: 扩写未完成章节（85%进度）
  ↓
步骤4: 生成过渡句（100%进度）
```

### 3. 查看结果

生成完成后，每个小说将显示：
- ✅ 小说标题和状态
- 📖 完整章节列表
- 📝 每章的大纲、正文和过渡句
- 📊 字数统计

## 技术架构

### 前端
- **框架**: Next.js 16 + React 19
- **UI库**: shadcn/ui + Tailwind CSS 4
- **类型安全**: TypeScript

### 后端服务
- **模型**: RWKV 7 (7.2B)
- **API**: 本地部署或远程服务
- **并发**: Promise.all 实现真并发

### 核心特性
```typescript
// 并行生成示例
const novels = await Promise.all(
  Array(parallelCount).map(() => 
    generateNovelOutline(genre, chapters)
  )
);

// 章节并行生成
const chapters = await Promise.all(
  novel.chapters.map(chapter => 
    generateChapterContent(chapter)
  )
);
```

## API 配置

默认配置在 `src/services/index.ts`：

```typescript
{
  baseUrl: "http://192.168.0.12:8000/v1/chat/completions",
  password: "rwkv7_7.2b",
  max_tokens: 10240,
  temperature: 1.0,
  // ... 其他固定参数
}
```

如需修改 API 地址或密码，可以创建自定义实例：

```typescript
import { RWKVService } from '@/services';

const customService = new RWKVService({
  baseUrl: "http://your-api-url:8000/v1/chat/completions",
  password: "your-password"
});
```

## 优化建议

### 并行数量选择
- **1-2个**: 适合测试和调试
- **3-5个**: 平衡速度和质量
- **6-8个**: 追求最大速度（需要强大的 API 服务）

### 章节数量建议
- **短篇**: 5-10章
- **中篇**: 15-30章
- **长篇**: 30-50章

### 质量优化
- 使用具体的小说类型（如"修仙玄幻"而不是"玄幻"）
- 设置合理的章节数，避免过多导致质量下降
- 适当的并行数量可以提供更多选择

## 常见问题

**Q: 生成速度慢？**
A: 检查 API 服务性能，调整并行数量，确保网络连接稳定。

**Q: 内容质量不佳？**
A: 尝试更具体的类型描述，减少并行数量以提高单个质量。

**Q: 章节内容重复？**
A: 这是 RWKV 模型的特性，可以通过调整温度参数优化。
